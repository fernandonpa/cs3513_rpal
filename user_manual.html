Overview

This RPAL (Right-reference Parallel Applicative Language) interpreter is a Python-based implementation of the RPAL language processor. It provides a complete pipeline for lexical analysis, parsing, standardization, and execution of RPAL programs.

RPAL is a functional programming language with features like higher-order functions, pattern matching, and tuple manipulation. This interpreter converts RPAL source code into executable instructions and runs them on a CSE (Control-Stack-Environment) machine.

Installation

  1. Clone the repository:

    git clone https://github.com/fernandonpa/cs3513_rpal.git
cd cs3513_rpal

  2. Requirements:

    • Python 3.6 or higher
    • No additional dependencies required

Usage

Basic Execution

To run an RPAL program:

python myrpal.py path/to/program.txt

Example:

python myrpal.py examples/Q1.txt

Command-line Options

The interpreter provides several command-line options:

Option Description
-ast Print the Abstract Syntax Tree and exit
-sast Print the Standardized Abstract Syntax Tree
-pretty Format tuple output as a clean, comma separated list

Examples:

# Display the Abstract Syntax Tree
python myrpal.py examples/Q1.txt -ast

# Display the Standardized Abstract Syntax Tree
python myrpal.py examples/Q1.txt -sast

# Format output tuples as clean, comma separated lists
python myrpal.py examples/Q3.txt -pretty

Using the Makefile

If you're using Unix/Linux/WSL, you can use the provided Makefile:

# Execute an RPAL program
make run file=examples/Q1.txt

# Print the Abstract Syntax Tree
make ast file=examples/Q1.txt

# Print the Standardized Abstract Syntax Tree
make sast file=examples/Q1.txt

# Format output in a pretty, comma separated manner
make pretty file=examples/Q3.txt

# Clean build artifacts
make clean

# Display help information
make help

RPAL Language Features

Syntax Basics

RPAL is a functional language with the following key features:

  1. Variables and Assignment:

    let x = 5 in x + 3

  2. Functions:

    let f x = x * 2 in f 5

  3. Conditionals:

    x gr 0 -> 'positive' | 'non-positive'

  4. Recursion:

    let rec fact N = N eq 0 -> 1 | N * fact(N-1) in fact 5

  5. Tuples:

    (1, 2, 3)

Common Programming Patterns

Variable Definition

let Variable = Value in Expression

Function Definition

let FunctionName Parameter1 Parameter2 = Expression in Body

Conditional Expressions

Condition -> TrueExpression | FalseExpression

Multiple Definitions (Where Clause)

Expression
where
   Definition1
   Definition2

Recursion

let rec FunctionName Parameters = Expression in Body

Advanced RPAL Features

Higher-Order Functions

RPAL supports higher-order functions, which means functions can accept other functions as arguments and return functions as results.

let map F L = L eq nil -> nil | (F(hd L)) aug (map F (tl L))
in map (fn x.x*2) (1 aug (2 aug (3 aug nil)))

Pattern Matching

RPAL allows pattern matching for function parameters:

let Split L = L eq nil -> nil, nil | hd L, tl L in
Split (1 aug 2 aug nil)

Closures

Functions in RPAL capture their lexical environment, forming closures:

let MakeAdder N = fn x . x + N in
let Add5 = MakeAdder 5 in
Add5 10

Tuple Operations

RPAL provides several built-in operations for working with tuples:

  • Order: Get the size of a tuple
  • Null: Check if tuple is empty
  • Stem: Get the first element
  • Stern: Get all but the first element
  • Aug: Add an element to a tuple
let T = (1,2,3) in
Order T, (Stem T), (Stern T)

CSE Machine Architecture

The CSE Machine (Control-Stack-Environment Machine) is the virtual machine that executes standardized RPAL programs. It consists of:

Components

  1. Control (C): A sequence of instructions to be executed
  2. Stack (S): Stores intermediate results during computation
  3. Environment (E): Maintains variable bindings during execution

Execution Phases

  1. Tokenization: Source code → Token stream
  2. Parsing: Token stream → Abstract Syntax Tree (AST)
  3. Standardization: AST → Standardized Tree (ST)
  4. Execution: ST → Program output via CSE Machine

Control Structures

The CSE Machine processes several control structures:

  • Delta: Function definitions
  • Lambda: Anonymous functions
  • Gamma: Function application
  • Beta: Environment markers
  • Operators: Built-in operations (+, -, *, /, etc.)

Built-in Functions Reference

RPAL includes several predefined functions:

Arithmetic

  • +, -, *, /: Standard arithmetic operations
  • **: Exponentiation
  • neg: Negation

Comparison

  • eq, ne: Equality and inequality
  • gr, ge: Greater than, greater than or equal
  • ls, le: Less than, less than or equal

Logical

  • &, or, not: AND, OR, NOT operations

Type Testing

  • Isinteger: Tests if value is an integer
  • Isstring: Tests if value is a string
  • Isboolean: Tests if value is a boolean
  • Istuple: Tests if value is a tuple
  • Isfunction: Tests if value is a function

String Operations

  • Conc: Concatenate strings
  • Stern: All but first character
  • Stem: First character

Tuple Operations

  • Order: Size of tuple
  • Null: Test for empty tuple
  • Augment (or aug): Add element to tuple

I/O

  • Print: Output values

Examples

Basic Arithmetic

let x = 10 in
let y = 5 in
x + y

Fibonacci Sequence

let fib n = n eq 0 -> 0 
            | n eq 1 -> 1 
            | fib(n-1) + fib(n-2)
in fib 10

Factorial Function

let rec factorial n = 
    n eq 0 -> 1 
    | n * factorial (n-1)
in 
print(factorial 5)

List Manipulation

let rec map f lst = 
    lst eq nil -> nil
    | f(hd lst) aug (map f (tl lst))
in
let square = fn x . x * x in
let result = map square (1 aug (2 aug (3 aug nil))) in
print(result)  // Outputs: (1,4,9)

Higher Order Functions

let rec fold f acc lst =
    lst eq nil -> acc
    | fold f (f acc (hd lst)) (tl lst)
in
let sum = fold (fn x y . x + y) 0 in
let product = fold (fn x y . x * y) 1 in
print(sum (1 aug (2 aug (3 aug (4 aug nil)))))  // Outputs: 10
print(product (1 aug (2 aug (3 aug (4 aug nil)))))  // Outputs: 24

Error Handling

Common errors you might encounter:

  1. Syntax Error: Check your code for missing parentheses, incorrect operators, etc.
  2. Undefined Variable: Make sure all variables are defined before use.
  3. Type Error: RPAL doesn't have static typing, but operations must be performed on compatible types.

Troubleshooting

  1. Program doesn't compile:
    • Check for syntax errors in your RPAL code
    • Verify that all variables are properly defined
    • Ensure matching parentheses and correct operator usage
  2. Unexpected output:
    • Use the -ast flag to verify the program structure
    • Use the -pretty flag for tuple outputs to see sorted values
  3. Interpreter crashes:
    • Check if the input file exists and is readable
    • Verify that the RPAL code is syntactically correct

Project Structure

  • src/Lexer: Tokenizes input text
  • src/Parser: Converts tokens to AST
  • src/tree_normalizer: Standardizes the AST
  • src/cse_machine: Executes the standardized AST

Best Practices

Performance Optimization

  • Avoid excessive recursion depth for large computations
  • Use tail recursion when possible to avoid stack overflow
  • Consider using accumulators in recursive functions to improve performance

Code Organization

  • Use meaningful variable names
  • Break complex expressions into smaller, named components
  • Use where clauses for local definitions that improve readability

Debugging Strategies

  • Use the -ast flag to verify your program structure
  • Insert Print statements at strategic points
  • Test complex functions with simple inputs first
  • Build programs incrementally, testing each component

Common Pitfalls

  • Missing parentheses around function arguments
  • Forgetting to handle base cases in recursive functions
  • Type mismatches in operations (trying to add a string and a number)
  • Shadowing variable names unintentionally
